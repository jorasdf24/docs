## **Tech stack**

- **Web application**: Next.js 15 with the App Router, React 19, and TypeScript for strongly typed front-end development.
- **API layer**: tRPC 11 combined with Zod schemas and TanStack Query for type-safe procedures and client-side data fetching/caching.
- **Authentication & email**: Better Auth (with GitHub and Google providers) plus Resend for transactional email delivery.
- **Database**: Drizzle ORM targeting a Postgres/Neon database, with migrations driven by `drizzle-kit`.
- **Background Jobs: **Inngest
- **Billing**: Paddle client and server SDKs for checkout, plan management, and webhook processing.
- **UI & styling**: Main use if ShadCN component library, Tailwind CSS, Radix UI primitives, and supporting component libraries (Lucide icons, DiceBear, etc.).
- **Tooling**: ESLint, TypeScript, and Tailwind CLI for developer experience.

## Project structure

```
/meetai
├── README.md
├── drizzle.config.ts
├── package.json
├── src/
│   ├── app/
│   │   ├── (auth)/
│   │   ├── (dashboard)/
│   │   └── api/
│   ├── components/
│   ├── constants/
│   ├── contexts/
│   ├── db/
│   ├── hooks/
│   ├── inngest/
│   ├── lib/
│   ├── modules/
│   ├── paddle/
│   └── trpc/
└── public/
```

- `src/modules/*` groups feature domains such as agents, meetings, checkout, and settings, each with server-side procedures and UI layers.
- `src/paddle` centralises Paddle API helpers, hooks, and webhook handling.
- `src/inngest` contains background jobs for transcript processing and AI summarisation.
- `src/trpc` defines the API routers, middleware (`protectedProcedure`), and React provider wiring.
- `src/app/api` exposes Next.js route handlers for auth, Paddle webhooks, Stream webhooks, Inngest, and tRPC.
